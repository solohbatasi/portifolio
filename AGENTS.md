# Project Engineering Rules

- Read the existing architecture before changing it.
- Do not introduce unnecessary dependencies.
- Use small, reusable Vue components.
- Keep business content separate from presentational components.
- Use semantic HTML.
- Ensure keyboard accessibility and visible focus states.
- Respect `prefers-reduced-motion`.
- Check mobile layouts at 360px, 390px and 430px widths.
- Check tablet and desktop layouts.
- Never expose credentials, API keys or confidential client information.
- Do not invent project performance figures, client testimonials or years of experience.
- Do not use percentage-based skill bars.
- Do not describe unfinished projects as production systems.
- Run linting and `npm run build` after substantial changes.
- Report files changed, commands executed and any remaining issues.
- Do not automatically push to GitHub.
- Do not overwrite deliberate manual changes without reviewing them.
